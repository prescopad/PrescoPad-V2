PrescoPad 2.0 — Full Codebase Audit Report
A clinic management app (doctor + assistant roles, queue, prescriptions, wallet billing, AI consultation transcription). Stack: FastAPI + Motor/MongoDB Atlas backend, React Native / Expo + Zustand + SQLite frontend, Groq Whisper + LLaMA 4 Scout for AI.

Below is the consolidated audit — what works, how it works, the issues found, and concrete improvements.

PART A — WHAT WORKS & HOW IT WORKS
A1. Backend (FastAPI)
Subsystem	How it works	Status
App bootstrap (backend_python/app/main.py)	FastAPI app with lifespan that connects Motor on startup, closes on shutdown. Global exception handler maps Mongo "pool paused" / autoreconnect to 503. CORS middleware reads ALLOWED_ORIGINS.	OK
Auth (OTP + JWT) (routes/auth.py, services/auth_service.py)	Phone-based OTP (6 digits, bcrypt-hashed, 5-min TTL). Demo mode returns fixed 123456. JWT access (7-day) + refresh (30-day) using HS256. require_doctor() / require_assistant() role guards.	Works, security gaps below
Data (patients / queue / prescriptions / custom medicines) (routes/data.py, services/data_service.py)	All routes correctly filter by clinic_id. Prescription IDs are human-readable RX-YYYYMMDD-XXXXXXXX. Soft deletes via is_deleted. Composite indexes on (clinic_id, ...).	Mostly OK
Wallet (services/wallet_service.py)	Per-user wallet, credit / debit / transaction log. ₹1 debited on prescription finalize.	Broken atomicity
Connection (doctor ↔ assistant) (services/connection_service.py)	Doctor invites by phone OR shares doctor_code; assistant joins; clinic_id propagated on accept.	Works, edge cases
Transcription (services/transcription_service.py)	Multipart audio → Groq Whisper large-v3-turbo → LLaMA-4-Scout diarization → LLaMA extraction (diagnosis, medicines, lab tests, advice, follow-up) → auto-registers custom medicines/tests for the clinic → persists transcript. Blocking Groq calls offloaded to thread pool.	Pipeline works, hardening needed
Analytics (services/analytics_service.py)	Aggregates by period (today/week/month): prescription count, earnings, top medicines/tests.	Broken $lookup
Notifications (services/notification_service.py)	Get-pending + mark-read endpoints exist.	No producer — dead code
A2. Frontend (Expo)
Subsystem	How it works	Status
App init (App.tsx)	2s splash + DB init in parallel, then RootNavigator.	Missing restoreSession call
Navigation (src/navigation/)	RootNavigator switches AuthStack ↔ DoctorTabs ↔ AssistantTabs based on isAuthenticated and isProfileComplete. Doctor heartbeat polls every 60s; AppState listener pauses on background.	Works
Auth screens (screens/auth/)	Landing → Login (role pick) → OTP → Registration (if new). Phone & OTP input is validated.	Resend OTP missing, clinic picker missing
Doctor screens (screens/doctor/)	Dashboard (queue + filters), Consult → MedicinePicker / LabTestPicker → PrescriptionPreview → RxSuccess. AI flow: AITranscription records via expo-av, uploads to backend, atomically writes draft via single updateDraft(...).	Works, edge cases
Assistant screens (screens/assistant/)	Dashboard + AddPatient + PatientSearch + PatientDetail. Doctor-ready indicator polls every 30s.	Works
API client (services/api.ts)	Axios instance, JWT injected on every request, 401 → refresh → retry. 10s default timeout (120s overridden for transcription).	Refresh race, no global mutex
State (Zustand) (store/)	Domain-scoped stores; tokens persisted to SecureStore.	restoreSession never invoked; polling leaks
SQLite (database/database.ts, seed.ts)	WAL mode; seeds 127 medicines + 71 lab tests on first run; PRAGMA user_version.	No real migrations; non-idempotent seed
PDF / share (services/pdfService.ts, shareService.ts)	expo-print HTML → PDF, expo-sharing for WhatsApp/SMS. SHA-256 hash via expo-crypto for tamper detection.	Mostly OK
PART B — ISSUES (RANKED BY SEVERITY)
B1. 🔴 CRITICAL — fix before any production traffic
#	Issue	Where	Why it matters
C1	Wallet deduction not atomic. Read-check-write pattern: read balance → if >= cost → write balance - cost. Two concurrent finalizes can both pass the check and over-debit; a concurrent recharge silently overwrites the debit.	wallet_service.py:40-64, data_service.py:263-296	Financial corruption, overdrafts, lost revenue.
C2	OTP brute-force is completely open. 6-digit OTP, no attempt counter, no per-phone rate limit, no lockout. 10⁶ space is trivial to enumerate in parallel.	auth_service.py:86-103, otp.py	Anyone can take over any account.
C3	No file-size limit on /api/transcription/analyze. await audio.read() slurps the whole multipart body into memory.	routes/transcription.py:18-57	One 1 GB upload = OOM crash of the worker.
C4	Hardcoded admin credentials in DB seed. 9999999999 / Admin@123 are created on every fresh startup.	config/database.py:99-101	Trivial admin takeover in any deployment that forgets to disable.
C5	OTP_DEMO_MODE = True by default. Anyone who reads 123456 is logged in.	config/settings.py:17-18	Auth bypass if shipped as-is.
C6	restoreSession() is never called on app boot. User is forced to log in on every cold start even though tokens are in SecureStore.	App.tsx, store/useAuthStore.ts:51-74	Broken UX + perceived logout — but more importantly the persisted refresh token is rendered useless.
C7	Token-refresh race in axios interceptor. Two parallel requests hit 401, both enter the refresh branch; whichever finishes second overwrites tokens. No mutex.	services/api.ts:29-66	Random logouts and corrupted token state.
B2. 🟠 HIGH — correctness / data integrity
#	Issue	Where
H1	Analytics $lookup joins on incompatible types. transactions.wallet_id is stored as str(wallet["_id"]) while wallets._id is ObjectId → lookup returns empty. Earnings/debit totals always 0.	analytics_service.py:76-88, wallet_service.py:28
H2	Queue token number race. count_documents(...) + 1 is not atomic; two assistants adding at once can mint duplicate tokens.	data_service.py:146-148
H3	Prescription finalize is not idempotent. Network retry after a successful finalize raises "already finalized" instead of returning the original result; clients with retries see failure.	data_service.py:263-296

H5	Notifications system is dead. Endpoints exist; no service ever inserts into notification_jobs.	services/notification_service.py
H6	Transcripts not linked to prescriptions. Audio is transcribed, extraction stored, but no prescription_id on the transcript document — audit trail cannot tie the consultation to the issued Rx.	transcription_service.py:351-363
H7	Frontend wallet deduction is optimistic with no rollback. useWalletStore sets local balance from response, but no rollback if a later step (PDF gen, finalize) fails.	store/useWalletStore.ts:34-45

PART C — IMPROVEMENT SUGGESTIONS (PRIORITIZED)
Sprint 1 — Security & money correctness (do not ship without these)
Atomic wallet ops — replace every read-then-write with a conditional $inc:

res = await db.wallets.update_one(
    {"user_id": user_id, "balance": {"$gte": amount}},
    {"$inc": {"balance": -amount}}
)
if res.modified_count == 0:
    raise ValueError("Insufficient balance")
Wrap (balance update + transaction insert) in a Motor session transaction.
OTP hardening — otp_attempts counter on the user doc; max 5 verify attempts per code; per-phone rate limit (e.g. 5 OTP requests / hour). Default OTP_DEMO_MODE=False. Validate Fast2SMS response.
Transcription upload guard — read Content-Length; reject > 25 MB; stream to a tempfile.NamedTemporaryFile; pass the path to Groq; finally: os.unlink(path).
Move admin seed credentials to .env and skip entirely when NODE_ENV=production.
Rate limiting middleware — slowapi or hand-rolled Redis token bucket; wire RATE_LIMIT_* settings.
Tighten CORS — refuse to start when origins is * and credentials are allowed.
Token-refresh mutex on the client — single in-flight refresh promise shared by all 401-victims:

let refreshing: Promise<void> | null = null;
...
refreshing = refreshing ?? doRefresh();
await refreshing;
refreshing = null;
Call restoreSession() before mounting RootNavigator in App.tsx.
Sprint 2 — Data integrity
Fix analytics $lookup — store transactions.wallet_id as ObjectId (and migrate existing rows), or $convert in the pipeline.
Queue tokens via counter collection — findOneAndUpdate({clinic_id, date}, {$inc: {seq:1}}, upsert=true).
Idempotent finalize — if status == "finalized", return success with the existing payload instead of erroring.
Link transcripts — add prescription_id and queue_item_id columns to transcripts.
Wallet rollback on prescription failure — on PDF/finalize error, call a credit-back API; UI shows a "Refunded ₹1" toast.
Server-validated balance check — frontend canAfford is hint-only; the atomic $inc is the source of truth.
Sprint 3 — UX & robustness
Resend OTP — countdown timer; reuse sendOtp from authService.
RegistrationScreen — wire the orphaned clinic-picker JSX or remove the styles; add regNumber format validation if your jurisdiction has one.
Mount NetworkBanner once in RootNavigator; surface "no connection — will retry" with auto-retry queue.
ConsultScreen discard guard — Alert.alert("Discard changes?", …) in beforeRemove.
PrescriptionPreviewScreen recharge return path — listen to wallet balance change and toast "Balance updated — tap Sign & Issue".
Clear setInterval in queue/heartbeat stores on unmount; expose explicit start()/stop() with refcounting.
Replace paddingTop: 50 with useSafeAreaInsets() across all screens.
Centralize snake_case ↔ camelCase: one util, used by every service; validate runtime with zod.
Surface error states from stores — every store gets error: string | null and clearError(); screens render a banner instead of swallowing.
Replace logo base64 in SecureStore with file-system storage (SecureStore has a 2 KB value limit on Android in some versions).
Sprint 4 — Performance & ops
Batch transcription lookups — fetch all clinic medicines once, build a Map<lowerName, doc> in-memory.
Bounded thread pool for Groq blocking calls — ThreadPoolExecutor(max_workers=8).
Add indexes: transactions(wallet_id, created_at), prescriptions(clinic_id, patient_id, created_at), TTL on notification_jobs.created_at (30 days).
Seed in a single transaction: db.execAsync('BEGIN; ... ; COMMIT;') — 10× faster cold start.
Streaming SHA-256 for PDFs (cryptoService.ts) — current code reads the whole base64 into memory.
useMemo the filtered queue in AssistantDashboard; convert transactions list to FlatList.
Sentry / Crashlytics + structured backend logging with phone-number redaction.
Sprint 5 — Architecture (longer term)
HIPAA-grade local storage: switch from expo-sqlite to expo-sqlite-encrypted (SQLCipher) with a key derived from the user's PIN + SecureStore.
TLS pinning in production builds via react-native-ssl-pinning or the new expo-network cert pinning.
Server-driven prescription numbering + signed PDF (server-side render + signature) so the device cannot generate phantom prescriptions.
Producer-side notifications — emit notification_jobs rows on: queue add, prescription finalize, low wallet balance. Wire a worker that pushes via FCM/APNs.
Schema migrations — proper versioned migration steps in database.ts instead of user_version ladder; integration tests for upgrade paths.
API contract test — generate TypeScript types from FastAPI's OpenAPI (openapi-typescript) so the snake_case/camelCase drift dies.
Doctor-code rotation endpoint.
Soft-leave for assistants — disconnect keeps historical data attached to clinic but flags the assistant as inactive.
PDF cloud archive — upload signed PDFs to S3/Cloudflare R2 + store URL on the prescription doc; survives device uninstall.

TL;DR
The app is well-architected and most flows work. The dangerous holes are concentrated in five places:

Wallet math is not atomic — both backend race conditions and frontend optimistic updates.
OTP auth has no brute-force protection and ships in demo mode by default.
Transcription upload has no size cap and the Groq pipeline has no retries/limits.
Auth session handling on the client — restoreSession never runs, and the refresh interceptor races itself.
Analytics $lookup joins ObjectId↔string so financial dashboards silently show zeros.
Fixing those five items closes ~80 % of the real risk. The rest of the list is hardening, UX polish, and documentation cleanup.
